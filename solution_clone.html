조달제품

Ark for Oracle

자체 DB복제 엔진과 최적의 관리 툴의 강력한 솔루션

Ark for Oracle은 DB 변경데이터를 실시간 전송/복제하는 솔루션으로서, 운영 시스템으로부터 실시간 Redo Log를 추출하여, 로그기반의 동일한 대기 DB를 구성합니다. 운영시스템의 부하를 최소화하고, 시스템의 성능을 최대화하는 현존하는가장 심플하고 안정적이며 비용 효율적인 솔루션입니다.

  • 운영시스템 중지 시 업무연속 보장

    • 작업 시작 후 10분이내 DB 서비스 복구 가능
    • 계획된 유지보수 작업시 대기 DB서버로 서비스 전환

  • 완벽한 DB데이터 정합성 보장

    • Oracle에 최적화된 실시간 DB복제 엔진
    • 완벽한 DB 데이터 정합성 보장
    • 디스크 오류 및 사용자 실수로부터 대기 DB 보호
    • 시스템 오류, 사용자 실수로 인한 손실된 데이터를 신속히 복구 (Log 지연적용 기능)

  • 실시간 DB 복제 엔진

    • 솔루션 자체 실시간 Log 전송 기반의 DB에 최적화된 복제엔진
    • 전송된 Log 지연 적용 기능
    • 적용된 Log 관리 기능 (삭제 및 보관)

  • 뛰어난 성능 및 효율성

    • DB의 Log 데이터만 복제하여 시스템 및 네트워크 부하 최소화
    • 전송 데이터 오류 체크 및 자동 재전송
    • 대기 DB를 조회, 출력, 테이프 백업 등을 위해 활용 가능

  • 구축 및 관리 용이

    • GUI 기반의 통합 관제시스템을 통한 손쉬운 관리
    • 장애 및 재해 상황 사전 감지를 위한 다양한 모니터링 데이터 제공
    • 운영시스템의 중단 없이 대기시스템 구축 가능
    • Archive Log 관리(자동삭제/보관) 기능

주요기능

  • 쉽고 유연한 업무 전환 기능

    • 서버 역할 전환 시에 IP 자동 전환 기능
    • Switchover 기능
      : 계획된 운영 시스템 정지시 대기 DB 서버를 통한 업무 지속
    • Failover 기능
      : 긴급 상황 발생시 외부 엔지니어 도움없이 대처 가능
    • DownTime 최소화
      (구축/이관 : 0 , Failover : 5M, Switchover : 10M)

  • 강력한 통합관리 기능

    • CLI & GUI 방식을 통한 통합 모니터링 기능
      (Process 상태 및 복제 현황 분석)
    • Mobile, E-mail 과의 연동으로 실시간으로 장애 통보 기능

  • 다양한 데이터 동기화 기능

    • 1:1, 1:N, N:1 등 다양한 복제 구성 지원
    • 초기복제 및 자동동기화 기능 제공
    • 로컬 및 원격지 백업 및 복구, DR구축 적용

기대효과

  • 비용효율적 구축

    • 저가용 Storage로 구성 가능
    • 대기시스템의 조회 업무로의 활용 가능

  • 복구시간

    • 재해 발생시 서비스 복구시간 대폭 감소
    • 장애 발생시 신속·정확한 복구 가능 (10분이내)
    • 최소한의 복구시간으로 영업 손실 최소화

  • 운영비용 절감

    • 기존 회선에서 증설 없이 네트워크 (TCP / IP)활용
    • 비용 효율적 도입, 유지보수 비용 절감

  • 시스템 효율 개선

    • 운영 시스템 부하 경감
    • 최소한의 데이터 전송으로 I/O감소

이기종 DBMS 복제솔루션(Ark for CDC)

GS인증
조달제품

Ark for CDC

이기종 DBMS간의 실시간 데이터 복제 솔루션

Ark for CDC는 서로 다른 OS와 DB환경에서 빠르고 안전하게 데이터를 복제하는 실시간 이기종 DB복제 솔루션 입니다. Direct log access 방식으로 Transaction log file에 직접 액세스하여 소스서버 운영 성능에 부하를 주지 않고 변경된 데이터만 추출합니다. 실시간으로 추출된 데이터는 SQL로 변환하여 Target DBMS에 전송되어 플랫폼 환경과 상관없이 이기종 DB 복제가 가능합니다.

  • Hybrid Platform

    • 다양한 OS와 DB지원
    • 이기종 DBMS간 데이터 복제
    • On-Premise와 Cloud 간의 데이터 복제

  • Direct Log Access

    • 데이터베이스 Redo Log 파일 직접 접근하여 빠른 데이터 추출
    • 파일 포맷을 분석하여 직접 변경 데이터 추출하여 데이터 일관성 보장

  • Middleware Architecture

    • Middleware를 통한 소스시스템의 부하 최소화
    • Tracing File 백업 가능
    • 표준화되고 효율적인 이기종 환경으로의 실시간 1:N 데이터 복제

주요기능

  • 이기종 DB 복제

    • 주요 Source DB 지원
    • 주요 Target DB 지원
    • 1:1, 1:N, N:1 등 다양한 복제 구성 지원
    • Oracle RAC, ASM, Multitenant 환경 지원

  • 데이터 안정성 & 정합성 보장

    • 전송 데이터 암호화
    • 데이터 정합성 체크 및 복구 아키텍처
    • 데이터 정합성 비교 기능

  • 고성능

    • Direct Log Access 방식
    • Network 부하 최소로 원거리 복제 가능
    • M/W 아키텍처를 통한 소스 서버 부하 최소화

  • 통합 모니터링 & 관리기능

    • CLI / GUI 방식 모니터링 기능
    • Process 상태 및 복제 현황 분석
    • Mobile 서비스 연동

기대효과

  • 비용 절감 및 효율 개선

    • 시스템 구축 및 운영 시 낮은 총소유비용(TCO) 추가구매 없이 데이터 Compare, Recovery 및 모니터링 기능 사용가능
    • IT 인프라 Structure 에 대한 부하를 최소화하며 비용 대비 효과적인 확장 방안 제공
    • Downtime 최소화와 부대비용 절감

  • 위험성 최소화 및 비즈니스 연속성 보장

    • 데이터 손실 최소화 및 데이터 정합성 보장
    • 변경 데이터를 통한 데이터 복구 및 추가 데이터 백업 방안 제공
    • 마이그레이션 및 업그레이드 중 최대 99%의 가용성 보장

  • 비즈니스 효율 증가

    • 실시간 복제 데이터를 활용한 타겟 서버의 활용 증가
    • 정확한 비즈니스적 판단을 돕기 위해 필요한 데이터 실시간 제공
    • 시스템 가용성 증가로 인한 고객 만족도 증대
    • 신속한 기술지원으로 수준 높은 이슈 대응 및 장애지원 보장

파일복제 솔루션(Ark for FR)

GS인증
조달제품

Ark for FR

Point in time copy 방식 파일 복제 솔루션

Ark for FR(File Replication)은 Point in time copy 방식 파일 복제 솔루션으로서 운영 서버 상의 데이터파일, 프로그램파일, 컨텐츠 파일 및 각종 문서 파일 등에 대하여 주기적으로 변경된 부분만을 대기 서버의 디스크로 복제하는 기능을 제공합니다. 주기적으로 서버간 파일 동기화를 수행함으로써, 기업의 중요한 데이터를 보호합니다.

  • 컨텐츠 복제

    서버간 웹 어플리케이션
    컨텐츠, 프로그램 파일을 동기화 및 복제

  • 재해 복구
    및 고가용성

    서버간 웹 어플리케이션
    컨텐츠, 프로그램 파일을 동기화 및 복제

  • 디스크 백업

    서버 데이터를
    별도의 디스크에 백업
    1:1, 1:N, N:1, N:N 으로 변경

  • 서버 이전

    전산 센터 이전 또는
    개별 서버 이전을 위한 데이터 복제

  • 자료 배포

    로컬 서버간 또는
    지역적으로 분산된 서버간 자료 배포

주요기능

  • Ark for FR Agent

    • 자체 프로토콜을 사용하여 두 파일에 대한 차이만을 복제
    • 사용자 설정에 의한 복제 주기 설정 지원
    • Link, Device, Owner, Group, Permissium 등 O/S 지원 모든 파일 형식 및 속성 복제 지원
    • 특정 디렉토리 및 특정 확장자 파일에 대한 복제 예외지원
    • Source node와 Backup node간의 기능 전환 지원

  • Ark for FR Manager

    • Source 및 Backup 복제 환경에 대한 모니터링 및 관리
    • Source 및 Backup 복제 상황에 대한 모니터링
    • Source 및 Backup 복제에 대한 리포팅

기대효과

  • 구축 및 운영 비용 최소화

    • Storage 기종에 구애 받지 않음
    • 운영 시스템의 중단 없이 구축 가능
    • 변경된 블록만을 전송하여 최소 네트워크 대역폭 사용을 통한 회선비용 절감

  • 시스템 부하 최소화

    • 대기 시스템 활용으로 운영시스템 부하 경감
    • 최소한의 데이터 전송으로 I/O 감소

  • 재해 발생 시 영업손실 최소화

    • 운영시스템의 재해 및 장애 발생시 신속한 복구 가능
    • 재해 발생시 복구시간 대폭 절감
